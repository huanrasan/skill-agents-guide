Fundamentos

¿Qué son los Skill Agents?

Los Skills son comportamientos inteligentes que le inyectas permanentemente a tu Agente Antigravity. No son recetas de pasos, sino un nuevo modo de pensar y actuar ante un tipo de problema.

🧠

Comportamiento Cognitivo

Un Skill altera cómo razona el agente. Le enseña qué herramientas usar, qué rol adoptar y bajo qué condiciones activarse.

⚡

Activación Automática

El agente detecta orgánicamente cuándo aplicar cada skill según la naturaleza de la petición, sin que el usuario tenga que recordarlo.

🔧

Personalizables

Puedes crear Skills a medida para tu stack: un experto en React, un auditor de seguridad, un arquitecto de microservicios...

📂

Siempre Disponibles

Al guardar la skill en .agents/skills/[nombre]/SKILL.md, quedan activas para toda la vida del proyecto.

Conceptos Clave

Workflows vs Skills

Dos herramientas complementarias. Úsalas juntas para construir pipelines sofisticados de trabajo.

📋 Workflow

Receta Procedural

  • ✦ Pasos lineales y predefinidos
  • ✦ Se activa con un slash command (ej. /deploy)
  • ✦ Ideal para tareas repetitivas estandarizadas
  • ✦ El "botón de encendido" de un proceso
  • ✦ Vive en .agents/workflows/
Ejemplo de uso: /setup-db, /run-tests, /deploy-prod
+
✦ Skill Agent

Talento Cognitivo

  • ✦ Define cómo pensar ante un problema
  • ✦ Se activa orgánicamente o bajo petición
  • ✦ Ideal para comportamientos complejos y de alta especialización
  • ✦ El "cerebro" detrás de un proceso
  • ✦ Vive en .agents/skills/[nombre]/SKILL.md
Ejemplo de uso: react-expert, security-auditor, multi-ai
🔗

Trabajan Juntos

Un Workflow puede invocar una Skill. El Workflow define cuándo y qué hacer; la Skill define cómo hacerlo con excelencia. Esto crea pipelines de IA de nivel profesional.

Estructura

Anatomía de una Skill

Una Skill es una carpeta dentro de .agents/skills/ que contiene al menos el archivo SKILL.md. Este archivo tiene dos partes obligatorias.

📁 Estructura de Archivos
.agents/
skills/
mi-skill/
SKILL.md ← obligatorio
scripts/ ← opcional
examples/ ← opcional
otra-skill/
workflows/
mi-workflow.md
01

YAML Frontmatter

Metadatos de la skill: nombre, descripción y las herramientas que tiene permitido usar. El campo description es crítico — es lo que le permite al agente decidir cuándo activar esta skill de forma automática.

02

Instrucciones en Markdown

El cuerpo del documento, escrito en Markdown natural. Aquí defines el comportamiento paso a paso, los roles que debe adoptar el agente, las herramientas que debe priorizar y las reglas que nunca debe romper.

# React Expert Skill

## Comportamiento al Activarse

Cuando esta skill se activa, adoptas el rol de un
Senior Frontend Engineer especializado en React.

## Reglas de Código

- Siempre usa componentes funcionales con hooks
- Usa `useCallback` y `useMemo` para optimizar renders
- Prefiere Zustand sobre Redux para estado global
- Aplica lazy loading en rutas con `React.lazy()`

## Revisión Automática

Antes de entregar código, siempre valida:
1. ¿Hay prop drilling? → Refactorizar con Context
2. ¿Los efectos tienen cleanup? → Agregar return
3. ¿Las listas tienen key única? → Validar `key` prop

# Security Auditor Skill

## Tu Rol
Eres un Penetration Tester y Auditor de Seguridad
OWASP. Tu trabajo es ENCONTRAR problemas.

## Checklist Obligatorio (OWASP Top 10)
1. Inyección SQL / NoSQL
2. Broken Authentication
3. XSS (Cross-Site Scripting)
4. Secretos expuestos en código
5. Dependencias con CVEs conocidos

## Proceso de Revisión

### Paso 1: Escanea el código con `grep_search`
buscando patrones de riesgo (eval, exec, innerHTML)

### Paso 2: Revisa dependencias usando `run_command`
→ `npm audit` / `pip audit`

### Paso 3: Genera reporte en artefacto con:
  - Nivel de severidad por hallazgo (CRITICAL/HIGH/MED)
  - Línea de código exacta afectada
  - Remediación sugerida

# Flujo Secure Deploy

## Instrucciones al Agente

Este Workflow combina múltiples Skills para garantizar
que el código sea seguro antes de llegar a producción.

## Paso 1: Auditoría de Seguridad
Activa y ejecuta tu Skill `security-auditor` sobre
todos los archivos modificados en el último commit.
Si hay hallazgos CRITICAL, detén el proceso.

## Paso 2: Tests Automáticos
// turbo
Ejecuta los tests del proyecto:
`npm run test -- --coverage`
Asegúrate de que la cobertura sea > 80%.

## Paso 3: Deploy
Solo si los pasos anteriores pasan sin errors:
`npm run build && npm run deploy:prod`

## Paso 4: Notificación
Llama a `notify_user` con un resumen del deploy.
Caso de Uso Avanzado

Skill Multi-AI Pipeline

El patrón más poderoso: el agente adopta automáticamente el perfil cognitivo óptimo para cada fase del proyecto, sin intervención manual.

El agente selecciona el perfil más adecuado según el tipo de proyecto y la fase activa.

💻

Implementación

Verificación
